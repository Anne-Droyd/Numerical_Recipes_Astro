\subsubsection{Romberg Integration} The Romberg integration method was used to numerically evaluate the integrals. This method progressively refines estimates of the integral by combining results from the trapezoid rule at different levels. It takes the following parameters:

\begin{itemize} \item \texttt{func}: The function to be integrated. \item \texttt{a} and \texttt{b}: The limits of integration. \item \texttt{n\_init}: The initial number of steps in the trapezoidal rule. \item \texttt{m}: The number of refinement levels. \item \texttt{tolerance}: A threshold for the precision of the result. \end{itemize}

The integration proceeds by building a triangular matrix, where the first column is the initial trapezoidal approximation, and each subsequent column improves the estimate. The process stops when the difference between successive estimates falls below the given tolerance. We then return the estimate of the integral given by the Romberg method.

\begin{lstlisting}[language=Python]
def romberg_integration(func, a, b, n_init, m, tolerance):
	# need to understand this.....
	int_matrix = np.zeros((m,m))
	n = n_init
	for col in range(m):
		for row in range(m - col):

			if col == 0:
				int_matrix[row][col] = trapezoid_rule(func,a,b,n)
				n *= 2
			else:
				int_matrix[row][col] = (4 ** col * int_matrix[row + 1, col-1] - int_matrix[row, col-1]) / (4 ** col-1)
				# https://www.integral-calculator.com/checked result against this.
				# https://en.wikipedia.org/wiki/Romberg%27s_method checked method against this but found it gave the wrong result

					# don't know if this is correct maybe it should be [0][col]

					return int_matrix[row][col]
	return int_matrix[0,-1]

\subsubsection{Trapezoidal Rule} The trapezoidal rule is used to calculate the initial estimate for the integrals. This rule is applied to the function \texttt{func} over the interval [a,b] with n points. If the first few points form a constant function, we use a special case to calculate the integral. For higher precision, we solve a quadratic equation based on the points and their corresponding function values. We then return the result of the trapezoidal rule as the integral.

\begin{lstlisting}[language=Python]
def trapezoid_rule(func,a,b,n):
	x = np.linspace(a,b,n)
	step_size = (b - a) / (n - 1)
	# quadratic of func(x) for 3 points after the start to estimate func(x[0])
	y = func(x[1:4])


	if isinstance(y,int):
		# hack to integrate constant.... think this actually might be the right way to do it....
		integral = y * x[-1] - y * x[0]
		return integral
	c = solve_quadratic(x[1:4],y)
	integral = step_size * (0.5 * c + sum(func(x[1:-1])) + 0.5 * func(x[-1]))

	return integral

\end{lstlisting}

\subsubsection{Solving the Quadratic} The function \texttt{solve\_quadratic} performs the substitution needed to find the constant C, the point where the parabola crosses the y-axis, allowing us to apply the trapezoidal rule effectively. The equation for this comes from the work outlined in \cite{socratic_quadratic_function}.

\begin{lstlisting}[language=Python]
def solve_quadratic(x,y):
	# https://socratic.org/questions/given-three-points-0-3-1-4-2-9-how-do-you-write-a-quadratic-function-in-standard
    #simultaneous equations to remove c, then sub a into b, once b is obtained, sub b into get a, then a and b into one of the original equations to get c
	b = (-1 * ((y[0] - y[1]) / ((x[0]) ** 2 + (x[1]) ** 2)) + y[2] - y[1]) / (( - x[0] + x[1]) / (x[0] ** 2 - x[1] ** 2) + x[2] - x[1])
	a = (y[0] - y[1] - b * (x[0] - x[1])) / (x[0] ** 2 - x[1] ** 2)
	c = y[2] - a * x[2] ** 2 - b * x[2]
	zero_point = c
	return zero_point
\end{lstlisting}

\subsubsection{Getting the CDF}

First up we're going to look at the first function called prob\_calc. First we'll notice that the pdf is generated for all x values in x\_list using list comprehension. We then calculate the integral using the Romberg integrator shown previously so that we are able to calculate the CDF.
We get the cumulative sum of the pdf values multiplied by the step size all divided by the total integral.

we then return the CDF and PDF for use later.

\begin{lstlisting}[language=Python]
def prob_calc(a,b,c,x_max,N_sat,A):
	# Create x values
	x_list = np.linspace(0, x_max, 5000)

	# Compute PDF at all points
	# Not sure if the PDF is supposed to be the integral over the sphere, but it works out nicely if it is
	# list comprehension faster than for loop somehow will try use it more
	pdf_values = np.array([q1_eq1(a, b, c, N_sat, x, A) * x ** 2 for x in x_list])
	# Compute full integral once using Romberg
	total_integral = romberg_integration(lambda x: q1_eq1(a, b, c, N_sat, x, A) * x ** 2, 0, x_max, 100, 10, 0.000001)

	# Use CUMSUM to construct the CDF
	# Am I allowed to use CUMSUM, surely so it's a simple command, I would say it is more simple than np.hist
	cdf_values = np.cumsum(pdf_values) * (x_list[1] - x_list[0]) / total_integral
	# ensuring edge case of 99.9 - 100 not ignored
	cdf_values[-1] = 1

	return cdf_values, pdf_values
\end{lstlisting}

\subsubsection{Random Number Generator}
The key component from the previous function is the \texttt{random\_number\_generator} function, so we will focus on that in detail. The goal was to implement a hybrid random number generator by combining multiple techniques: two separate \textbf{Linear Congruential Generators (LCGs)} feeding into two \textbf{64-bit XOR generators}, which are then combined into a \textbf{Multiply-With-Carry (MWC) generator}.

Each sub-generator must maintain its own state, so I set the states as global variables to ensure they persist across function calls. Initially, these states are set to \texttt{None}, allowing them to be assigned a seed value later.

For the seed values, I chose arbitrary "random" numbers without applying a specific selection method. These seed values are assigned only once and remain unchanged throughout execution.

\subsubsection*{Linear Congruential Generators (LCGs)}
For the two LCGs, I needed to define the parameters $a$ and $b$. Following the same approach as with the seeds, I selected these values arbitrarily. Additionally, I set $m$ (the modulus) to \textbf{64 bits}, which defines the upper limit for the generated numbers.

However, since the state values are also \textbf{64-bit integers}, this configuration causes inherent \textbf{overflow issues} due to nonzero, positive parameters. As a result, the LCGs naturally loop back on themselves. I initially suspected that this might cause a bias toward lower values, but after benchmarking against a \textbf{32-bit modulus}, I observed that the LCGs still produced a uniform distribution and did not noticeably affect the results.

\subsubsection*{64-bit XOR Generators and Multiply-With-Carry (MWC)}
Next, I set the parameters for the \textbf{64-bit XOR generators}, using well-tested values from \cite{marsaglia2003xorshift}. Additionally, I chose a prime number for the MWC generator, as it is preferable for ensuring randomness quality.

Once the setup was complete, I performed the \textbf{XOR operations} on the states for the \textbf{64-bit XOR generators}. After generating four random numbers from the sub-generators, I applied XOR operations to two pairs of them. The resulting values were then passed into the \textbf{MWC generator}, which produced the final random number.

\subsubsection*{Normalization of the Output}
To scale the generated number within a specified range $[\text{low}, \text{high}]$, I normalized the MWC output by dividing it by $2^{64}$, effectively converting it into a fraction between 0 and 1. This fraction was then scaled to the desired range using:

\begin{equation}
\text{random value} = \left( \frac{\text{MWC output}}{2^{64}} \right) \times (\text{high} - \text{low}) + \text{low}
\end{equation}

This ensures that the final output is uniformly distributed within the requested range.

def spherical_to_cartesian(r, theta, phi, center_point):

\subsubsection{Sorting}

The merge\_sort function iteratively splits the arrays into single cells, then once the single cells have been generated, they are passed onto the merge function.

\begin{lstlisting}[language=Python]
def merge_sort(arr, column_to_sort):
	if len(arr) <= 1:
		# cells have been split into singles, return the cell
		return arr

	# iteratively split the array in two until one cell is left
	mid = len(arr) // 2
	left_half = merge_sort(arr[:mid], column_to_sort)
	right_half = merge_sort(arr[mid:], column_to_sort)

	# once all cells have been split merge them
	return merge(left_half, right_half, column_to_sort)
\end{lstlisting}

The merge function takes the arrays passed on from the previous function, and sorts based on lowest value until the while loop is unsatisfied. It starts by checking if the current value in the left array is smaller than the current value in the right array, and if so it appends it to the sorted list. If this condition is not satisfied then it appends the right value. Once a value has been appended the appropriate counter is increased. Should anything be left when the while loop has been exited then the remaining value is appended as this will have been sorted.

\begin{lstlisting}[language=Python]
# Does what it says on the tin init
def merge(left, right, column_to_sort):
	# Init
	sorted_arr = []
	i = j = 0

	# Merge two sorted halves
	# Use while loop
	while i < len(left) and j < len(right):
		# if left < right append left then restart loop increase i
		if left[i][column_to_sort] <= right[j][column_to_sort]:
			sorted_arr.append(left[i])
			i += 1
		# If right < left append right then restart loop increase j
		else:
			sorted_arr.append(right[j])
			j += 1

	# Append remaining elements from both halves
	# If theres anything left in the arrays it should be sorted as it would be the final item
	sorted_arr.extend(left[i:])
	sorted_arr.extend(right[j:])

	return sorted_arr
\end{lstlisting}

\subsubsection{Riddler's Method}
We've passed our parameters to Riddler's method, we start by initializing the matrix for storing the estimations. We then calculate the inital estimates in the first column by calling the central difference function and decreasing h as we go.
We then make more guesses based on the previous estimates, similar to Romberg method implemented earlier.
We also make a check for convergence to a tolerance value and if this is reached we exit with the top row last column value. otherwise we exit with the last value.

\begin{lstlisting}[language=Python]
def riddlers_method(x, h, m_num,tolerance):
	# ripped from previous tutorial
	# Init matrix
	diff_vals = np.zeros((m_num, m_num))

	# calculate initial colum with decreasing h
	for m in range(m_num):
		h = h * 1 / (2 ** m)
		diff_vals[m][0] = central_difference(x, h)

	# Go iteratively go through the rest of the matrix recalculating the diff value to obtain a better result
	for j in range(1, m_num):
		for i in range(0, m_num - j):
			diff_vals[i][j] = ((2 ** (2 * j)) * diff_vals[i + 1][j-1] - diff_vals[i][j-1]) / (2 ** (2 * j)-1)
		# Check for Convergence
		if abs(diff_vals[0, j] - diff_vals[0, j-1]) < tolerance:
			# This is just a randomly set tolerance but I did a bit of trial and error to decide on a value
			# but break if converging, Here is why I'm unsure of the integral analog being row-1,col
			return diff_vals[0, j]
	# Return last cell if succesfully made it out alive
	return diff_vals[0,-1]
\end{lstlisting}
